# Daily_Training

백준 자료구조/알고리즘을 “매일 3시간” 루틴으로 훈련하는 저장소입니다.

---

## 목표
- 매일 3시간 내:
  1) Python으로 당일 문제 1개 AC
  2) 전날 핵심 자료구조/문제를 C로 재구현(또는 로컬 테스트 통과)
- 자료구조 구현 감각(분기/예외/입출력/복잡도)을 빠르게 체화
- 설명보다 “제출/디버깅/최적화”에 도움이 되는 형태로 기록
- 단기간에 백준 골드 1~2 수준까지 상승
- 언어 비중: 기본 7:3(Python:C) → 단계가 올라갈수록 C 비중 증가

---

## 기본 루틴(총 3시간)
- 워밍업 15분: 전날 문제를 “코드 안 보고” 핵심 분기만 재작성(제출은 선택)
- Python 메인 70~90분: 당일 목표 문제 1개 AC
- C 45~60분: 전날 문제를 C로 재구현(배열/포인터/문자열 처리 등)
- 마무리 10분: 오늘 배운 실수/규칙 2줄 기록

---

## Done(하루 완료 기준)
- [ ] Python 1문제 AC(또는 실패해도 원인/수정 포인트 기록)
- [ ] C 재구현 코드가 로컬에서 최소 테스트 통과
- [ ] `note.md`에 “실수/규칙 2줄 + 내일 1순위 1줄” 작성

---

## 폴더 구조 규칙
- `DayNN(MM.DD)` 폴더에 그날 코드/노트 저장 (Day 번호는 훈련 회차 기준)
- 각 Day 폴더에는 가능한 한 아래 파일을 둡니다:
  - `문제번호_설명.py` (또는 `.c`)
  - `note.md` (짧게라도 반드시)

예)
- `Day01(2.17)/10828_스택.py`
- `Day02(2.19)/10845_큐.py`
- `Day02(2.19)/note.md`

---

## 빠른 규칙(제출 안정성)
- Python: 입력은 `sys.stdin.readline`, 출력은 리스트에 모아서 `"\n".join()`
- C: 경계조건(빈 스택/빈 큐) 처리 우선, 문자열 비교는 `strcmp`
- 막히면: “막힌 지점 1문장 + 시도한 것 + 코드”를 남기고 피드백 요청
